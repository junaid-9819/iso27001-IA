Day 4: Technological Controls (Core IT)

This guide provides a comprehensive set of questions and a structured approach for Day 4 of your ISO 27001:2022 internal audit. Today's focus is on assessing core IT security controls.

Controls Covered: A.8.1–A.8.20 (15 controls prioritized, with flexibility to expand).

Attendees: IT team, system administrators, ISMS owner.

Core Technological Controls (A.8)

A.8.1 User Endpoint Devices

  • Are user endpoint devices (e.g., laptops, phones) secured with encryption and anti-malware?
  • How are security policies enforced on these devices (e.g., MDM)?
  • Can you show me a sample device configuration or compliance report?
  • Are personal devices (BYOD) allowed, and if so, how are they secured?
  • How are endpoint security violations detected and addressed?

A.8.2 Privileged Access Rights

  • Who has privileged access to systems, and how is it assigned?
  • Are privileged activities logged and reviewed regularly?
  • Can I see a list of privileged users and a sample log of their actions?
  • How often are privileged rights reviewed, and by whom?
  • Is there evidence of unauthorized privilege use being caught?

A.8.3 Information Access Restriction

  • How is access to information restricted based on business needs (e.g., role-based)?
  • Are access controls applied consistently across systems?
  • Can you provide the access control policy and a sample permissions list?
  • How are access violations detected and corrected?
  • Is there a process for handling access exceptions?

A.8.4 Access to Source Code

  • Where is source code stored, and who has access to it?
  • Are changes to source code logged and approved?
  • Can I see the repository access controls and a change log?
  • How is unauthorized access to code prevented?
  • Are developers trained on secure code access practices?

A.8.5 Secure Authentication

  • Is multi-factor authentication (MFA) or strong passwords enforced for system access?
  • How are authentication credentials (e.g., passwords) stored and protected?
  • Can you show me authentication settings or an MFA deployment report?
  • Are weak authentication methods (e.g., SMS) avoided?
  • How are failed authentication attempts monitored?

A.8.6 Capacity Management

  • How is system capacity monitored to ensure security and availability?
  • Are there alerts for capacity overloads or performance issues?
  • Can I see capacity monitoring tools or a recent capacity report?
  • Who is responsible for capacity planning, and how is it documented?
  • Are capacity-related security risks (e.g., DoS) addressed?

A.8.7 Protection Against Malware

  • What anti-malware tools are deployed across systems and endpoints?
  • How often are malware definitions updated and scans performed?
  • Can you provide a recent malware scan result or update log?
  • Are malware incidents tracked and resolved?
  • How are staff trained to recognize and report malware?

A.8.8 Management of Technical Vulnerabilities

  • How are technical vulnerabilities identified (e.g., scans, penetration tests)?
  • Is there a process to prioritize and remediate vulnerabilities?
  • Can I see the latest vulnerability scan report and remediation plan?
  • Who oversees vulnerability management, and how often is it reviewed?
  • Are critical vulnerabilities addressed within a defined timeframe?

A.8.9 Configuration Management

  • Are secure configuration baselines defined for systems and devices?
  • How are configurations audited for compliance?
  • Can you show me a baseline config and an audit result?
  • Are configuration changes tracked and approved?
  • How are misconfigurations detected and corrected?

A.8.10 Information Deletion

  • What process ensures sensitive data is securely deleted when no longer needed?
  • Are deletion tools or methods (e.g., wiping, shredding) used?
  • Can I see a deletion policy or log from a recent data removal?
  • How is deletion verified to ensure no data remnants?
  • Are third parties involved in deletion compliant with security standards?

A.8.11 Data Masking

  • Is data masking applied to sensitive information (e.g., in test environments)?
  • How is masking implemented, and what tools are used?
  • Can you show me an example of masked data or a masking policy?
  • Who decides when and where masking is required?
  • Is masking effectiveness tested or audited?

A.8.12 Data Leakage Prevention

  • What measures prevent unauthorized data exfiltration (e.g., DLP tools)?
  • Are data transfers monitored for suspicious activity?
  • Can I see a DLP policy or a report of blocked transfers?
  • How are data leakage incidents investigated and resolved?
  • Are staff aware of data leakage risks and controls?

A.8.13 Information Backup

  • Are backups of critical data and systems scheduled and performed?
  • How are backups stored (e.g., off-site, encrypted)?
  • Can you provide a backup schedule and a recent test result?
  • Are backups tested for restorability, and how often?
  • Who is responsible for backup management?

A.8.14 Redundancy of Information Processing Facilities

  • Are critical systems designed with redundancy (e.g., failover servers)?
  • How is redundancy tested to ensure availability?
  • Can I see a redundancy plan or test log?
  • Are single points of failure identified and mitigated?
  • Is redundancy aligned with business continuity needs?

A.8.15 Logging

  • What system events (e.g., logins, changes) are logged across IT assets?
  • How long are logs retained, and where are they stored?
  • Can I see a sample log from a critical system?
  • Are logs protected from tampering or unauthorized access?
  • Who reviews logs, and how are anomalies handled?

A.8.16 Monitoring Activities If Time Permits

  • What tools or processes monitor system activity for anomalies?
  • How are monitoring alerts prioritized and resolved?
  • Can I see a monitoring dashboard or alert log?
  • Are monitoring responsibilities assigned and documented?
  • Is monitoring effectiveness reviewed periodically?

A.8.20 Networks Security If Time Permits

  • Are networks protected with firewalls, segmentation, or intrusion detection?
  • How are network security measures monitored and tested?
  • Can I see a network diagram and recent test results (e.g., pen test)?
  • Who manages network security, and are they trained?
  • Are network vulnerabilities addressed promptly?

Execution Tips for Day 4

  • Time Management: Allocate ~25-30 minutes per control (15 controls ≈ 6-7 hours). Start with endpoints and access (A.8.1–A.8.5), then move to operations and networks.
  • Evidence Focus: Collect configs, logs, and reports (recent, e.g., within 6-12 months). Request live demos if possible (e.g., MFA login).
  • Approach: Ask technical questions with IT staff, then verify with "Can I see…?" Be prepared for hands-on system reviews.
  • Prioritization: If time runs short, focus on A.8.1, A.8.2, A.8.7, A.8.8, and A.8.15—core operational controls. Defer A.8.16–A.8.20 to Day 5 if needed.
  • Wrap-Up: Debrief with the IT lead (15-20 minutes) to confirm findings and preview Day 5 (Technological wrap-up and compliance).
