Day 5: ISO 27001:2022 Audit Questionnaire (Technological II & Wrap-Up)

Focus Area: Technological Controls (A.8.21–A.8.34) & Organizational Controls (A.5.24–A.5.37) & Wrap-Up

Objective: Collaboratively assess development-related technical controls, incident management, compliance processes, and finalize the audit with a review of findings and next steps.

Personnel Needed: IT developers, compliance officer, ISMS owner, top management

Total Questions: 70 (38 for Technological + 32 for Organizational, averaging 2-3 per control)

Morning Session: Technological Controls (A.8.21–A.8.34)

Security of Network Services

A.8.21
  1. How do you secure network services (e.g., VPN, DNS)? Can we review the service security policy together (e.g., network service policy)?
  2. Are security requirements defined for these services? Can we discuss examples (e.g., service configs)?
  3. How is service security monitored? Can we see monitoring methods (e.g., service logs)?

Segregation of Networks

A.8.22
  1. How are networks segmented (e.g., VLANs)? Can we review the segmentation policy or diagrams (e.g., network maps)?
  2. How is segmentation enforced? Can we discuss enforcement methods (e.g., firewall rules)?

Web Filtering

A.8.23
  1. How is web access filtered to prevent threats? Can we review the web filtering policy (e.g., filtering rules)?
  2. What tools are used, and how are they set up? Can we see examples (e.g., filter configs)?

Use of Cryptography

A.8.24
  1. How is cryptography used to protect data? Can we review the cryptography policy (e.g., encryption policy)?
  2. Can we discuss examples of encrypted data or keys (e.g., key management logs)?
  3. How are cryptographic keys managed? Can we see management practices (e.g., key records)?

Secure Development Life Cycle

A.8.25
  1. How is security integrated into the SDLC? Can we review the SDLC policy (e.g., development process)?
  2. Can we discuss a recent project with security steps (e.g., project docs)?
  3. How are developers trained on security? Can we see training evidence (e.g., training records)?

Application Security Requirements

A.8.26
  1. How are security requirements defined for applications? Can we review requirement documents (e.g., app security specs)?
  2. How are these requirements implemented? Can we discuss examples (e.g., app configs)?

Secure System Architecture and Engineering Principles

A.8.27
  1. Are secure architecture principles applied? Can we review the principles or policy (e.g., architecture guidelines)?
  2. Can we discuss a system designed with these principles (e.g., system design docs)?

Secure Coding

A.8.28
  1. How is secure coding enforced? Can we review the secure coding policy (e.g., coding standards)?
  2. Can we discuss examples of code reviews or guidelines (e.g., review reports)?
  3. How are coding flaws addressed? Can we see fix examples (e.g., bug fix logs)?

Security Testing in Development and Acceptance

A.8.29
  1. How is security testing conducted during development? Can we review the testing process (e.g., test plan)?
  2. Can we discuss recent test results (e.g., test reports)?
  3. How are test failures resolved? Can we see resolution methods (e.g., fix records)?

Outsourced Development

A.8.30
  1. How is security ensured in outsourced development? Can we review the outsourcing policy or contracts (e.g., dev contracts)?
  2. How is outsourced work monitored? Can we discuss monitoring methods (e.g., audit reports)?

Separation of Development, Test, and Production Environments

A.8.31
  1. How are dev, test, and prod environments separated? Can we review the separation policy (e.g., env separation rules)?
  2. Can we see evidence of separation (e.g., network configs)?
  3. How are breaches detected? Can we discuss detection methods (e.g., breach logs)?

Change Management

A.8.32
  1. How are changes to systems managed? Can we review the change management process (e.g., change procedure)?
  2. Can we discuss recent changes (e.g., change logs)?
  3. How are security impacts assessed? Can we see examples (e.g., risk assessments)?

Test Information

A.8.33
  1. How is test data protected? Can we review the test data policy (e.g., test data rules)?
  2. Is real data avoided in testing? Can we discuss examples (e.g., test data samples)?

Protection of Information Systems During Audit Tests

A.8.34
  1. How are systems protected during audits or tests? Can we review the protection policy (e.g., audit protection rules)?
  2. Can we discuss protections from recent tests (e.g., test logs)?
